§01 · Architecture

Browser to server to sidecar to CLI.

Two paths through the stack. Results stream over Redis for durability. Keystrokes ride a direct WebSocket for sub-10 ms echo.

§05 · Install

Sixty seconds to a running dashboard.

Two paths. Pick the one that matches the box you have lying around.

Local · Docker Compose

recommended for trying it out
bash 
git clone https://github.com/kr4t0n/argus
cd argus
cp .env.example .env
docker compose up -d

Brings up Postgres, Redis, the server, and the web app on localhost:3000. The bundled stack ships a sidecar on the host so you can spin a Claude Code or Codex agent immediately.

Cluster · Helm

production deploys
bash 
helm repo add argus https://kr4t0n.github.io/argus/helm
helm repo update
helm install argus argus/argus \
  --set postgres.url=$POSTGRES_URL \
  --set redis.url=$REDIS_URL

BYO Postgres and Redis, or run them in-cluster. Full chart docs in helm/argus/README.md.

Adding remote machines

One sidecar per box. systemd or launchd.

Each remote box runs a single argus-sidecar binary. Hand it a server URL and a join token, wrap it in a unit file, and it self-registers as a Machine on first dial-home. Recipes for systemd, launchd, and a one-line shell installer in the install guide.
